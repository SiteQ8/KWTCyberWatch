# Support

## How to Get Help

### Documentation

- [README.md](README.md) - Project overview, setup, and usage
- [CONTRIBUTING.md](CONTRIBUTING.md) - How to contribute
- [SECURITY.md](SECURITY.md) - Security policy and vulnerability reporting

### Asking Questions

- **GitHub Issues**: Open an issue with the `question` label for general questions
- **Discussions**: Use [GitHub Discussions](https://github.com/SiteQ8/KWTCyberWatch/discussions) for open-ended conversations

### Reporting Bugs

1. Search [existing issues](https://github.com/SiteQ8/KWTCyberWatch/issues) first
2. Use the **Bug Report** template when creating a new issue
3. Include reproduction steps, environment details, and expected behavior

### Feature Requests

- Open an issue using the **Feature Request** template
- Describe the problem you're trying to solve and your proposed solution

### Security Issues

**Do NOT report security vulnerabilities through public issues.** See [SECURITY.md](SECURITY.md) for responsible disclosure.

## Response Times

- **Critical security issues**: Within 48 hours
- **Bug reports**: Within 1 week
- **Feature requests**: Reviewed monthly
- **General questions**: Best effort

## Community

- Follow our [Code of Conduct](CODE_OF_CONDUCT.md)
- Be respectful and constructive in all interactions
